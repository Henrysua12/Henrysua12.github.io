The Game of Life

C++ Data Structures

The Game of Life is a cellular automaton game that has 0 players. Invented by mathematician John Conway. Originally a player would input the inital state however, this version works a bit different. This is smaller version of Conway's game of life built in C++ with a big emphasis on arrays and loops. Instead a user input what generation it wants to see from the original state and it will produce the generation.

For this version of Conway's game of life, I’ve confined it a board size of 10 by 20. Above you can see 4 images from the program each from generation mainly generation 1, 2, 3 and then generation 18,361 as you can see after a while cell have arranged in an odd pattern. The rules for Conway’s game of life are simple but can lead to some very interesting concepts. Which you can read more about here. Below are the rules to Conway’s game of life.

  1. If a live cell has fewer than 2 neighbors, It dies (underpopulation).
  2. If a live cell has more than 3 neighbors, It dies (overpopulation).
  3. If a dead cell has exactly 3 neighbors, It becomes alive (reproduction).

In this project I learned about Data Structures such as Arrays and how to use principles such as if statements and loops. The script works by running it with a value to indicate which generation you want displayed. If you gave it the value of 15 it would display the 15th generation of cells. The value of 0 would just display the base generation.
